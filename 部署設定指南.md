# 上線部署設定指南（Vercel + Supabase + Google 登入）

這份帶你把網站發佈到網路上，並加上「Google 登入 + 進度寫入資料庫」。
照順序做，遇到要填的值我都標好了。有卡住把畫面截圖給我。

> 重點觀念：`config.js` 裡的 anon key 是**公開金鑰**，可以放進 git、沒關係。
> 真正的安全由資料庫的 RLS 規則把關（使用者只能寫自己的 email、且必須是 @hanlin.com.tw）。

---

## 🔐 金鑰安全（給你和 RD 看）

金鑰分兩種，處理方式完全不同：

| 金鑰 | 種類 | 放哪裡 | 能不能外流 |
|---|---|---|---|
| Supabase **anon key**（config.js 那把） | 公開金鑰 | 前端、git、瀏覽器都看得到 | ✅ 設計上就公開，沒關係 |
| Supabase **service_role key** | 機密 | 只在 Supabase 後台，**絕不進程式/git** | ❌ 外流＝資料庫門戶洞開 |
| 資料庫 **密碼** | 機密 | 只在 Supabase 後台 | ❌ 不外流 |
| Google OAuth **Client Secret** | 機密 | 只貼在 Supabase 後台的 Google 設定 | ❌ 不進前端/git |

**為什麼 anon key 公開是安全的？**
它只是「這個 Supabase 專案的門牌」，本身沒有權限。真正決定「誰能讀寫什麼」的是資料庫的 **RLS 規則**（`supabase_schema.sql` 裡那幾條）：
- 使用者只能讀/寫**自己 email** 的資料，改不了也看不到別人的。
- 寫入時 email 還必須是 `@hanlin.com.tw`。
- 沒登入的人完全不能碰資料。

所以就算有人拿到 anon key，也只能做「登入後對自己資料」的操作，碰不到別人、也拿不到全表。這是 Supabase 官方的標準用法，前提就是「**每張表都有開 RLS**」——我們的表已經開了。

**你只要守住三件事：**
1. service_role key、資料庫密碼、Google Client Secret → 永遠只留在後台，不要貼進任何程式或訊息。
2. 每張新資料表都要記得開 RLS（目前只有一張，已開）。
3. GitHub repo 設 **Private**（雖然 anon key 公開沒事，但影片/內部教材不需要公開）。

## 步驟 1：把程式放上 GitHub

1. 到 https://github.com 登入（不確定有沒有帳號就用公司 Google 信箱試登入/註冊）。
2. 右上角 **+ → New repository**：
   - Repository name：`hanlin-training`（隨意）
   - 設 **Private**（內部用）
   - 不要勾任何 README/.gitignore（我們已經有檔案）
   - 按 **Create repository**
3. 在你電腦上，用「Git Bash」或終端機進到 **site** 資料夾，執行（把網址換成你剛建的 repo）：

```bash
cd "d:/AI工作區_業務教育訓練/site"
git init
git add .
git commit -m "翰林業務教育訓練平台"
git branch -M main
git remote add origin https://github.com/你的帳號/hanlin-training.git
git push -u origin main
```

> 之後你每次更新，只要 `git add . && git commit -m "更新" && git push`，Vercel 會自動重新發佈。

---

## 步驟 2：建立 Supabase（資料庫 + 登入）

1. 到 https://supabase.com → 用 GitHub 登入 → **New project**。
   - Name：`hanlin-training`
   - Database Password：自己設一組（記起來）
   - Region 選 **Northeast Asia (Tokyo)**（離台灣近）
   - 按 Create，等 1～2 分鐘建立完成。
2. 左側 **SQL Editor → New query**，把專案裡 `supabase_schema.sql` 的內容整段貼上 → **Run**。
   （這會建立 `progress_log` 資料表與權限規則。）
3. 左側 **Project Settings（齒輪）→ API**，記下兩個值：
   - **Project URL**（像 `https://abcd1234.supabase.co`）
   - **anon public** key（很長一串）

---

## 步驟 3：開 Google 登入

### 3-1 在 Google Cloud 建立 OAuth 憑證
1. 到 https://console.cloud.google.com → 上方選/建立一個專案。
2. 左側 **APIs & Services → OAuth consent screen**：
   - User Type：如果這個專案在公司的 Google Workspace 底下，選 **Internal**（這樣「只有 @hanlin.com.tw 能登入」由 Google 幫你擋，最保險）。
     若只能選 **External**，也沒關係——我們的程式和資料庫都會再擋一次網域。
   - 填 App name（例：翰林業務教育訓練平台）、你的信箱，其他預設，儲存。
3. 左側 **APIs & Services → Credentials → Create Credentials → OAuth client ID**：
   - Application type：**Web application**
   - **Authorized redirect URIs** 加入這一行（把網址換成你步驟 2 的 Project URL）：
     ```
     https://你的專案.supabase.co/auth/v1/callback
     ```
   - 建立後會給你 **Client ID** 和 **Client Secret**，先留著。

### 3-2 在 Supabase 啟用 Google
1. Supabase 左側 **Authentication → Providers → Google** → 打開 Enable。
2. 貼上剛剛的 **Client ID** 與 **Client Secret** → Save。

---

## 步驟 4：把金鑰填進網站

1. 打開專案的 `js/config.js`，把兩個值換成步驟 2 記下的：
   ```js
   SUPABASE_URL: "https://你的專案.supabase.co",
   SUPABASE_ANON_KEY: "你的 anon public key",
   ```
2. 存檔後推上 GitHub：
   ```bash
   git add js/config.js
   git commit -m "設定 Supabase 連線"
   git push
   ```

---

## 步驟 5：用 Vercel 發佈

1. 到 https://vercel.com → 用 GitHub 登入 → **Add New… → Project**。
2. 選你的 `hanlin-training` repo → **Import**。
   - Framework Preset：**Other**（純靜態）
   - 其他都預設 → **Deploy**。
3. 一分鐘後會給你一個網址，像 `https://hanlin-training.vercel.app`。**複製它。**

---

## 步驟 6：把 Vercel 網址回填（很重要，否則登入會失敗）

1. **Supabase → Authentication → URL Configuration**：
   - **Site URL** 填：`https://hanlin-training.vercel.app`
   - **Redirect URLs** 按 Add，加入：`https://hanlin-training.vercel.app/**`
   - （想在本機測試，再加一行 `http://localhost:3000/**`）
   - Save。
2. 完成！打開你的 Vercel 網址 → 應該看到登入畫面 → 用公司 Google 帳號登入 → 進入平台。

---

## 後台（看全部人的進度，只限管理者）

- 網址：在你的 Vercel 網址後面加 `/admin.html`
  （例：`https://hanlin-training.vercel.app/admin.html`）
- **誰能看**：只有被加入 `admins` 名單的 email。其他 @hanlin 帳號就算打開這個網址，也只會看到「你沒有後台檢視權限」。
- **怎麼把自己（或別人）設成管理者**：
  - 方法 A：執行 `supabase_schema.sql` 前，先把裡面那行
    `insert into public.admins ... '請改成你的email@hanlin.com.tw'` 換成你的公司信箱。
  - 方法 B（隨時新增）：Supabase → **Table Editor → admins → Insert row**，填入要開通的 email 即可。
- 後台功能：看全部人的 `email / 專案 / 步驟 / 完成時間`、搜尋、**匯出 CSV**（Excel 可開）。

---

## 驗收：資料有沒有進資料庫

- 隨便完成一個步驟後，到 **Supabase → Table Editor → progress_log**，
  就會看到一列：`usermail / project / step / completed_at`。
- 你在後台看得到**所有人**的紀錄；右上可 **Export → CSV** 匯出成 Excel。

---

## 之後的日常

- **更新內容/簡報/程式**：改完 → `git add . && git commit -m "更新" && git push`
  → Vercel 自動重新發佈，使用者重新整理就是新版（已設定不卡快取）。
- **本機預覽**（可選）：在 site 資料夾執行 `npx serve`（或 `python -m http.server 3000`），
  開 `http://localhost:3000`。注意本機測試要先在步驟 6 加入 localhost 那行。
